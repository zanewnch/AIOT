    AIOT Backend API Documentation - v0.0.0

    AIOT Express Application

    A Node.js Express application built with TypeScript following modern best practices.

    📁 Project Structure

    app/
    ├── src/                    # Source code
    │   ├── app.ts             # Express app configuration
    │   ├── server.ts          # Server entry point
    │   ├── routes/            # Route handlers
    │   │   ├── index.ts       # Home routes
    │   │   └── users.ts       # User API routes
    │   ├── controllers/       # Business logic
    │   │   └── userController.ts
    │   ├── middleware/        # Custom middleware
    │   │   └── errorHandler.ts
    │   └── types/             # TypeScript type definitions
    │       └── index.ts
    ├── public/                # Static assets
    │   ├── css/              # Stylesheets
    │   ├── js/               # Client-side JavaScript
    │   └── images/           # Images
    ├── views/                 # Template files (Jade/Pug)
    ├── dist/                  # Compiled TypeScript output
    └── Dockerfile.be          # Docker configuration

    🚀 Getting Started

    Prerequisites

    • Node.js (v18+ recommended)
    • npm or yarn

    Installation

    1. Install dependencies:
    npm install
    1. Development mode (with hot reload):
    npm run dev
    1. Build for production:
    npm run build
    1. Run production build:
    npm run serve

    📝 Available Scripts

    • npm start - Start with tsx (development)
    • npm run dev - Development mode with nodemon
    • npm run build - Compile TypeScript to JavaScript
    • npm run build:watch - Watch mode compilation
    • npm run serve - Run compiled JavaScript

    🛠 API Endpoints

    • GET / - Home page
    • GET /api/users - Get all users
    • GET /api/users/:id - Get user by ID

    🔧 Technologies Used

    • Node.js - Runtime environment
    • Express.js - Web framework
    • TypeScript - Type safety
    • Jade/Pug - Template engine
    • tsx - TypeScript execution
    • nodemon - Development auto-reload

    🏗 Development

    The project uses:

    • ES Modules ("type": "module" in package.json)
    • Strict TypeScript configuration
    • Organized folder structure following Node.js best practices
    • Error handling middleware
    • Type-safe API responses

    📚 API Documentation (Swagger)

    概念與架構

    本專案使用獨立 YAML 文件的方式來管理 Swagger API 文檔，與 TypeDoc 內部程式碼文檔完全分離：

    🔄 架構變更 (YAML 驅動)

    • TypeDoc: 內部程式碼文檔（npm run docs:generate）
    • Swagger: API 文檔（獨立 YAML 文件驅動）
    • 分離關注點: 程式碼文檔 vs API 規格文檔

    Backend 設定

    1. 獨立 YAML 文檔

      • 主要規格文件：/docs/swagger.yaml
      • 包含完整的 OpenAPI 3.0 規格定義
      • 所有 API 端點、模式、安全配置都在此文件中

    2. 配置檔案

      // /src/config/swaggerConfig.ts
      import yaml from 'js-yaml';
      import fs from 'fs';

      // 讀取獨立的 YAML 文檔
      const swaggerPath = path.join(__dirname, '../../docs/swagger.yaml');
      const swaggerDocument = fs.readFileSync(swaggerPath, 'utf8');
      const specs = yaml.load(swaggerDocument);

    3. API 端點

      • /api/swagger.json - 返回 OpenAPI spec JSON（供前端使用）
      • /api/docs - Swagger UI 頁面（備用方案）

    4. 依賴管理

      # 安裝 YAML 解析器
      npm install js-yaml @types/js-yaml

      # 移除舊的依賴（已不需要）
      npm uninstall swagger-jsdoc @types/swagger-jsdoc

    🎯 YAML 文檔結構

    # /docs/swagger.yaml
    openapi: 3.0.0
    info:
      title: AIOT API
      version: 1.0.0
      description: API documentation for AIOT project

    paths:
      /api/init/rbac-demo:
        post:
          tags: [Initialization]
          summary: Initialize RBAC demo data
          # ... 完整的端點定義

    components:
      schemas:
        User:
          type: object
          properties:
            id: { type: integer }
            username: { type: string }
            # ... 完整的模式定義

    Frontend 整合

    1. 前端組件

      • SwaggerDocPage.tsx - 完整的文檔頁面
      • 通過 API 調用獲取 /api/swagger.json

    2. 資料流程

      /docs/swagger.yaml → js-yaml 解析 → /api/swagger.json
      Frontend 調用 → swagger-ui-react 渲染

    📝 文檔維護方式

    1. 更新 API 文檔：

      • 直接編輯 /docs/swagger.yaml
      • 無需修改程式碼註釋
      • 重啟服務器即可看到變更

    2. 添加新端點：

      # 在 swagger.yaml 中添加
      /api/new-endpoint:
        get:
          tags: [NewFeature]
          summary: New endpoint description
          responses:
            '200':
              description: Success response

    使用方式

    • 前端整合：在 React 應用中使用 SwaggerDocPage 組件
    • 獨立訪問：訪問 http://localhost:8000/api/docs
    • JSON 規格：http://localhost:8000/api/swagger.json

    優點

    • 文檔與程式碼分離：更清晰的關注點分離
    • 易於維護：統一的 YAML 文件管理
    • 版本控制友好：YAML 格式更適合 Git diff
    • 團隊協作：非開發人員也可以編輯文檔
    • 工具整合：支援各種 OpenAPI 工具鏈

    TypeDoc vs Swagger

    • TypeDoc:

      • 用途：內部程式碼文檔、API 參考
      • 來源：TypeScript 註釋
      • 訪問：npm run docs:generate && npm run docs:serve

    • Swagger:

      • 用途：外部 API 文檔、API 測試
      • 來源：獨立 YAML 文件
      • 訪問：/api/docs 或前端整合

    🐳 Docker

    Build and run with Docker:

    docker build -f Dockerfile.be -t aiot-app .
    docker run -p 8000:8000 aiot-app
