    Class JwtAuthMiddleware

    Express 應用程式的 JWT 驗證中間件

    使用 JSON Web Tokens 提供驗證和授權功能。 支援必需驗證和可選驗證兩種模式。

    JwtAuthMiddleware

    Example

    const jwtAuth = new JwtAuthMiddleware();

    // 必需驗證
    app.use('/api/protected', jwtAuth.authenticate);

    // 可選驗證
    app.use('/api/public', jwtAuth.optional);

    authenticate

    • authenticate(req: Request, res: Response, next: NextFunction): Promise<void>

      必需的 JWT 驗證中間件

      驗證 JWT 令牌並確保使用者存在於資料庫中。 對於沒有有效令牌的請求，回傳 401 狀態碼拒絕存取。

      Parameters

      • req: Request

        Express 請求物件

      • res: Response

        Express 回應物件

      • next: NextFunction

        Express next 函數

      Returns Promise<void>

      Promise

      Throws

      需要存取令牌 - 當沒有提供令牌時

      Throws

      無效或過期的令牌 - 當令牌驗證失敗時

      Throws

      找不到使用者 - 當使用者不存在於資料庫中時

      Throws

      驗證失敗 - 當發生意外錯誤時

      Example

      app.get('/api/profile', jwtAuth.authenticate, (req, res) => {
        res.json({ user: req.user });
      });

    • optional(req: Request, res: Response, next: NextFunction): Promise<void>

      可選的 JWT 驗證中間件

      嘗試驗證 JWT 令牌，但即使令牌遺失或無效也會繼續執行。 僅在找到有效令牌和使用者時設定 req.user。

      從請求中提取 JWT 令牌

      優先檢查 Cookie 中的 JWT 令牌，然後回退到 Authorization 標頭。 支援 Authorization 標頭中的 Bearer 令牌格式。

      Parameters

      • req: Request

        Express 請求物件

      Returns null | string

      JWT 令牌字串，如果找不到則回傳 null

      Example

      優先順序：
      1. Cookie: `req.cookies.jwt`
      2. Authorization 標頭: `Bearer <token>`
